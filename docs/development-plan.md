## 开发规范

### Git 分支策略
- `main`: 主分支，稳定版本
- `develop`: 开发分支
- `feature/*`: 功能分支
- `bugfix/*`: 修复分支

### 提交规范
- `feat:` 新功能
- `fix:` Bug 修复
- `docs:` 文档更新
- `refactor:` 代码重构
- `test:` 测试相关
- `chore:` 构建/工具

### 代码规范
- 后端：阿里巴巴 Java 开发手册
- 前端：Vue 官方风格指南
- 使用 ESLint + Prettier 格式化

## 风险控制

### 技术风险
- **Sa-Token 集成复杂度**：预留 1 天缓冲时间
- **并行评审一致性**：使用乐观锁 + 事务保证
- **大文件上传**：MinIO 分片上传

### 进度风险
- **每日站会**：每天同步进度
- **风险预警**：延期超过半天立即上报
- **加班预案**：关键节点可适当加班

## 验收标准

### 功能验收
- 所有需求功能正常运行
- 无 P0/P1 级别 Bug
- 代码通过 Review

### 性能验收
- 页面加载时间 < 2s
- API 响应时间 < 500ms
- 支持 100 并发用户

### 文档验收
- API 文档完整
- 用户手册清晰
- 部署文档可操作
